## TL;DR

把 OpenClaw 接入微信，你的 AI 助手就能在微信里聊天了。用 `@canghe/openclaw-wechat` 插件，但要注意：需要买 API Key，要配代理服务，有封号风险，只适合学习和研究。

## 前置要求

开始之前，确认这些：

- [ ] 安装了 Node.js（18+）
- [ ] 安装了 OpenClaw：`npm install -g @openclaw/cli`
- [ ] 有微信个人号（企业号不行）
- [ ] 会用命令行（基础就行）
- [ ] 知道要花钱：API Key 和代理服务都是付费的
- [ ] 知道有风险：账号可能被封

## 核心步骤

### 步骤 1：了解微信插件（别看错文档了）

OpenClaw 的微信集成是靠插件实现的，不是官方自带。而且有两个不同的插件，搞错了会很麻烦。

#### 官方文档说的那个（过时了）

- 包名：`@icesword760/openclaw-wechat`
- 仓库：https://github.com/icesword0760/openclaw-wechat
- 状态：404，估计是没了

**别用这个！** 官方文档的信息已经过时了。

#### 实际存在的那个（最新、最活跃）

- 包名：`@canghe/openclaw-wechat`
- 仓库：https://github.com/freestylefly/openclaw-wechat
- Stars：1,463
- 最后更新：2026-03-19（今天！）
- License：MIT

**功能：**
- 私聊、群聊
- 文字、图片
- 扫码登录
- 多账号

#### 差异在哪

| 方面 | 官方文档的 | 实际的 |
|------|-----------|--------|
| 包名 | `@icesword760/...` | `@canghe/...` |
| 仓库 | 404，没了 | 活跃，今天刚更新 |
| 配置 | 简单 | 复杂，要 API Key、代理 |

**用哪个？** `@canghe/openclaw-wechat`，这个才是最新的。

### 步骤 2：安装插件

用对的名字安装：

```bash
openclaw plugins install @canghe/openclaw-wechat
```

**预期输出：**

```
✓ Plugin @canghe/openclaw-wechat installed successfully
```

**出问题了？**

```bash
# 看插件状态
openclaw plugins list

# 升级插件
openclaw plugins update wechat
```

### 步骤 3：买和配 API Key

**注意：** 这个插件不是免费的，要买 API Key！

#### 3.1 获取 API Key

**怎么搞：**
- 联系客服买
- 进群等（项目在优化，想体验先加群）
- 确认格式：`wc_live_xxxxxxxxxxxxxxxx`

**买之前确认好价格和使用限制！**

#### 3.2 配置 API Key

```bash
openclaw config set channels.wechat.apiKey "wc_live_xxxxxxxxxxxxxxxx"
```

### 步骤 4：配代理服务

**注意：** 代理服务是必填的！

```bash
openclaw config set channels.wechat.proxyUrl "http://your-proxy-server:3000"
```

**代理干嘛用的：**
- 让微信连接更稳定
- 处理协议细节
- 可能提供账号池

**选可靠的代理服务商！**

### 步骤 5：配 Webhook（云部署要，本地可跳过）

你在云服务器上部署的话，要配 webhook 来收消息。

```bash
# 设置公网 IP（云部署必填）
openclaw config set channels.wechat.webhookHost "your-server-ip"

# 设置端口（默认 18790）
openclaw config set channels.wechat.webhookPort 18790

# 设置路径（默认 /webhook/wechat）
openclaw config set channels.wechat.webhookPath "/webhook/wechat"
```

**本地测试的话，webhook 可以先不配。**

**云部署配置：**

```yaml
# ~/.openclaw/openclaw.json
channels:
  wechat:
    enabled: true
    apiKey: "wc_live_xxxxxxxxxxxxxxxx"
    proxyUrl: "http://your-proxy:3000"
    webhookHost: "1.2.3.4"
    webhookPort: 18790
    webhookPath: "/webhook/wechat"
    deviceType: "ipad"
```

**记得在云服务器防火墙里开放 webhookPort！**

### 步骤 6：启用微信频道

```bash
openclaw config set channels.wechat.enabled true
```

**检查配置：**

```bash
openclaw config get

# 应该看到：
# channels.wechat.apiKey: wc_live_xxx
# channels.wechat.proxyUrl: http://your-proxy:3000
# channels.wechat.enabled: true
```

### 步骤 7：首次登录

启动 Gateway，扫二维码：

```bash
openclaw gateway start
```

**首次启动会显示码：**

```
[INFO] WeChat plugin starting...
[INFO] Please scan the QR code to log in:

███████████████████
███████████████████
███████████████████

[INFO] Waiting for QR code scan...
```

**登录步骤：**
1. 微信打开"扫一扫"
2. 扫终端显示的二维码
3. 确认登录

**注意：**
- 二维码有效期就几分钟
- 显示出来赶紧扫
- 过期了就重启 Gateway 重新生成

### 步骤 8：测试一下

#### 8.1 测私聊

给自己或朋友发：

```
@OpenClaw 你好
```

**预期回复：**

```
你好！我是你的 AI 助手，有什么可以帮你的吗？
```

#### 8.2 测群聊

在微信群里 @ 机器人：

```
@OpenClaw 帮我总结下今天的工作
```

**预期回复：** 机器人会回复总结。

#### 8.3 看日志

```bash
openclaw logs --follow

# 预期输出：
# [INFO] WeChat message received: 你好
# [INFO] AI response generated: 你好！...
```

## 常见问题

### 问题 1：二维码过期

**现象：** Gateway 显示二维码，扫了说过期。

**原因：**
- 二维码有效期短（几分钟）
- 网络慢扫超时
- Gateway 没完全启动

**怎么搞：**

```bash
# 1. 确认网络稳
ping your-proxy-server

# 2. 重启 Gateway 重新生成码
openclaw gateway restart

# 3. 显示出来赶紧扫
# 微信先开好"扫一扫"

# 4. 看日志确认 Gateway 启动完成
openclaw logs --follow
```

**预防：**
- 网络要稳
- 提前开好微信"扫一扫"
- 盯着日志，确认码生成了

### 问题 2：收不到消息

**现象：** 能发，但 OpenClaw 不回，收不到微信消息。

**原因：**
- webhookHost 没配或配错了
- webhookPort 没开放
- 防火墙挡了

**怎么搞：**

```bash
# 1. 检查 webhook 配置
openclaw config get channels.wechat.webhookHost
openclaw config get channels.wechat.webhookPort

# 2. 确认端口开了（云服务器）
# 去云服务器控制台看安全组/防火墙

# 3. 测端口通不通
telnet your-server-ip 18790

# 4. 看日志确认 webhook 正常
openclaw logs --follow

# 预期输出：
# [INFO] Webhook server listening on port 18790
```

**注意：** 本地部署如果没配 webhook，可能收不到消息。

### 问题 3：Gateway 启不了

**现象：** `openclaw gateway start` 一下就退出或报错。

**原因：**
- 配置文件语法错了
- API Key 或 proxyUrl 配错了
- 代理服务连不上

**怎么搞：**

```bash
# 1. 检查配置文件语法
cat ~/.openclaw/openclaw.json | python3 -m json.tool

# 2. 验证配置
openclaw config get channels.wechat

# 3. 测代理服务连通性
curl http://your-proxy-server:3000

# 4. 看详细日志
openclaw logs --level debug

# 5. 重启 Gateway
openclaw gateway restart
```

### 问题 4：回复很慢

**现象：** 发了消息，OpenClaw 回得很慢或超时。

**原因：**
- 代理服务不稳
- 网络质量差
- 代理服务器负载高

**怎么搞：**

```bash
# 1. 查代理服务状态
# 联系客服确认

# 2. 测网络
ping your-proxy-server

# 3. 看日志定位问题
openclaw logs --follow

# 4. 考虑换个代理
# 如果一直不稳的话
```

### 问题 5：插件装不上

**现象：** `openclaw plugins install @canghe/openclaw-wechat` 报错。

**原因：**
- npm 网络问题
- 包名错了
- OpenClaw 版本太低

**怎么搞：**

```bash
# 1. 检查网络
ping registry.npmjs.org

# 2. 升级 OpenClaw
npm update -g @openclaw/cli

# 3. 确认包名对不对
# 应该是 @canghe/openclaw-wechat，不是 @icesword760/...

# 4. 清理 npm 缓存
npm cache clean --force

# 5. 重试安装
openclaw plugins install @canghe/openclaw-wechat
```

## ⚠️ 重要提醒

### 账号安全

1. **只能学习和研究** - 别干坏事，后果自负
2. **有封号风险** - 用非官方客户端就是有风险
3. **用测试号** - 别用重要的微信号
4. **别在重要群里用** - 避免不必要的风险

### 成本

1. **要买 API Key** - 不是免费的
2. **要代理服务** - 可能要额外花钱
3. **价格不透明** - 联系客服问清楚
4. **评估好成本** - 用之前确认好

### 技术要求

1. **配置复杂** - 要配好几个参数
2. **要云服务器** - 生产环境需要
3. **网络要稳** - 影响体验
4. **可能要调试** - 技术问题得自己排
