# 🤖 GitHub Issues 自动化收集快速开始

这个工具可以帮你快速从 GitHub 收集 OpenClaw 社区的问答，并自动转换为教程文章。

## 🚀 5 分钟快速开始

### 步骤 1: 配置 GitHub 仓库

打开 `scripts/collect-github-issues.js`，修改第 19-20 行：

```javascript
const CONFIG = {
  owner: 'openclaw',          // 仓库所有者
  repo: 'openclaw',           // 仓库名称 ⚠️ 修改为实际仓库
```

### 步骤 2: 设置翻译 API（可选）

#### 选项 A: DeepL（推荐，准确）

```bash
# 1. 注册免费账号: https://www.deepl.com/pro-api
# 2. 复制 API Key
export DEEPL_API_KEY=your_key_here
```

#### 选项 B: OpenAI

```bash
export OPENAI_API_KEY=your_key_here
```

#### 选项 C: 不翻译

跳过此步骤，只收集英文内容。

### 步骤 3: 运行收集

#### 默认配置（获取所有已关闭的 issues）

```bash
# 收集所有已关闭的 GitHub issues
npm run collect-issues
```

#### 限制数量

如果只想获取部分 issues，编辑 `scripts/collect-github-issues.js`：

```javascript
const CONFIG = {
  maxIssues: null,  // 获取所有（默认）
  maxIssues: 100,   // 只获取 100 个
  maxIssues: 1000,  // 只获取 1000 个
```

## 📋 工作流程

```
GitHub Issues
    ↓
收集脚本 (npm run collect-issues)
    ↓

    ↓
转换脚本 (npm run convert-issues)
    ↓

    ↓
网站构建 (npm run build)
```

## 🎯 使用技巧

### 只转换特定的 issues

编辑 `scripts/convert-issues-to-posts.js`：

```javascript
const CONFIG = {
  // 只转换这些 issues
  selectedIssues: [42, 123, 456], // 添加 issue 编号
```

### 调整收集数量

编辑 `scripts/collect-github-issues.js`：

```javascript
const CONFIG = {
  maxIssues: 50,  // 改为想要的数量
```

### 定期自动更新

#### GitHub Actions（推荐）

1. Fork 你的仓库到 GitHub
2. 添加 Secrets: `DEEPL_API_KEY`
3. 创建 `.github/workflows/collect.yml`：

```yaml
name: 每周收集 Issues
on:
  schedule:
    - cron: '0 2 * * 0'  # 每周日凌晨 2 点
  workflow_dispatch:

jobs:
  collect:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - uses: actions/setup-node@v3
        with:
          node-version: '22'

      - name: 收集 Issues
        env:
          DEEPL_API_KEY: ${{ secrets.DEEPL_API_KEY }}
        run: |
          npm run collect-issues
          npm run convert-issues

      - name: 提交更改
        run: |
          git config user.email "action@github.com"
          git config user.name "GitHub Action"
          git add src/content/
          git diff --quiet && git diff --staged --quiet || git commit -m "chore: 更新收集的文章"

      - name: 推送
        uses: ad-m/github-push-action@master
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
```

#### 本地定时任务

```bash
# 编辑 crontab
crontab -e

# 每周日凌晨 2 点运行
0 2 * * 0 cd /path/to/openclaw-tutorial && npm run collect-issues && npm run convert-issues
```

## 🛠️ 故障排除

### 404 仓库未找到

```bash
❌ 仓库 not found: openclaw/openclaw
```

**解决**: 修改脚本中的仓库名为实际的 OpenClaw 仓库。

### API 速率限制

```bash
❌ GitHub API 速率限制
```

**解决**:
```bash
export GITHUB_TOKEN=your_token
```

### 翻译失败

```bash
❌ 翻译失败: 401 Unauthorized
```

**解决**: 检查 API Key 是否正确。

## 💡 最佳实践

1. **定期收集**: 每周运行一次收集最新内容
2. **人工审核**: 自动转换后需要人工补充解决方案
3. **标签管理**: 使用 GitHub labels 帮助分类
4. **质量优先**: 选择有价值的 issues 转换，不是全部
